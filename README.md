# General Instructions

Before starting any assessment, please review the following:

- Each repository contains a detailed `README.md` with full instructions
- Ensure you follow the submission guidelines carefully
- Use any frameworks, libraries, or tools you are comfortable with
- You are allowed to use AI tools (ChatGPT, Claude, Copilot, etc.)

We are more interested in how you solve problems than in memorization of syntax.

---

# Submission Process

All submissions must be completed through the official application form.

You will be required to provide:

- GitHub repository link of your solution
- Setup and run instructions
- Short technical reflection answers

Make sure your repository is publicly accessible.

---

# Evaluation Criteria

Your submission will be evaluated based on:

- Functionality and correctness
- Code quality and structure
- Problem-solving approach
- System design decisions
- Error handling and edge cases
- Clarity of documentation

---

# Important Notes

- There is no single “correct” solution
- We value clean, maintainable, and well-reasoned implementations
- Partial submissions are accepted if clearly explained
- Focus on quality over quantity
